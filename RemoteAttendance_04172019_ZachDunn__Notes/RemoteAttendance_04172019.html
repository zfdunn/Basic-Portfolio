1.1-GIT_Hub

Purpose of github is to collaborate, with the rest of the tech world, sharing code. Rev tracking is huge plus side. Instead of sharing code through email, slack, text docs, etc...Git Hub allows for team collaboration, whether personal, professional; work team project allows for "rev safety", to understand when, where & who applied broken code, and mitigation of this broken code within the group; the plus side is if the group is all failing to mitigate the issue at hand, then the general public can assist. Be cautious here, as to what you "commit"

2.1-Repo Section (procedure)

When creating new repository, keep public, and always initialize w/ "README" file

Step 1 - "Clone/Download"; copy ssh to clipboard;

Step 2 - d/l to machine; "push" -- copy ssh key in git -- continue to terminal steps:

Step 3 - (using terminal) - ensure you "cd" into appropriate folder first!

2.1.1 - git clone (insert pasted ssh from clipboard)

2.1.2 - git add -A (track changes in repo)

2.1.3 - git status (lets you understand whether or not tracking cmd is working...and changes to be committed.)

2.1.4 - git commit -m (addition of the "-m" alllows you to tag the commit w/ a message)

2.1.5 - git push origin master (push my commit to github; share w/ team; push to gitcloud etc.)
